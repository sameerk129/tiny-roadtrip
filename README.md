# Tiny Procedural Roadtrip

A tiny lonely journey through infinite realities. An ambient procedural roadtrip simulation toy — *not* a game.

The car drives forever. The world keeps changing. You tweak. You watch. You drift.

## Run it

```bash
npm install
npm run dev
```

Then open [http://localhost:3000](http://localhost:3000).

That's the whole setup. No backend, no auth, no database, no API keys.

## What's inside

- **Next.js 15 + React 19 + TypeScript**, app router.
- **TailwindCSS** for the translucent dashboard overlays.
- **Framer Motion** for tiny UI fades only.
- **HTML Canvas 2D** for everything visual. No WebGL. No Three.js.
- **Web Audio API** for ambient hum, wind, rain, thunder, and procedural radio drones.

Everything else is hand-rolled procedural generation.

### Render loop

There is exactly one `requestAnimationFrame` loop, owned by `CanvasStage`. React state never re-mounts the canvas. The loop:

1. Reads the latest UI settings from a ref (`settingsRef.current`)
2. Mirrors them into the `world.settings` mutable object
3. Updates transitions (biome / weather / time blends)
4. Updates events, shooting stars, particles
5. Renders all layers
6. Applies post effects
7. Writes a `world` snapshot back to a ref so HUD / Radio can read at ~5 Hz

This means the canvas is decoupled from React rendering. Sliders update at 60fps without ever causing a canvas remount.

### Performance

- **Particle pool** is a fixed-size array (~600 slots). Particles are reused; nothing allocates during the hot loop.
- **Stars** are pre-generated once. Twinkle is a pure `Math.sin`.
- **Mountains** are deterministic noise sampled at 8px steps along the screen.
- **Midground scenery** is sampled per slot index — no spawn/despawn lists.
- **Audio** uses long-lived nodes; only gain ramps change.
- **No allocations** in `step()` for typical frames.
- DPR is capped at 2 to avoid 4K canvas explosions.
- The loop pauses while the tab is hidden (visibilitychange).

### Procedural systems

- **Biomes (18)**: each has a 3-stop sky gradient, three mountain layers, ground/road palette, accent neon, and a midground scenery type.
- **Weather (9)**: clear, rain, thunderstorm, snow, fog, ash, meteor shower, neon rain, sandstorm. Each defines particle kind, fog boost, road reflection, wind, visibility, and thunder triggers.
- **Time of Day (6)**: sunrise / noon / sunset / night / midnight / eclipse. Each tints + darkens the entire palette and controls star alpha + shooting star chance.
- **Events (15)**: UFO, giant creature, gas station, military convoy, lightning, static burst, meteor, ghost hitchhiker, portal, train, giant moon, road split, sky whale, distant city explosion, abandoned robot.
- **Apocalypse / Reality stability**: drives camera shake, sky bend, glitch slices, chromatic aberration bands, accent flickers, and detuned radio pads.

### Radio

A small synthesis bus with three sine/triangle voices retuned per station, plus filtered noise for static. Track titles are generated by combining word lists. DJ lines are sampled from per-station banter pools. **No copyrighted music.**

Stations: Neon Mirage (synthwave), Drift FM (lo-fi), Last Signal (post-apocalypse survival), EMG-7 (emergency broadcast), Static, Off.

## Controls

- **Right panel**: biome / weather / time of day / reality stability / cruise speed / seed re-roll.
- **Left panel**: vehicle toggles (headlights, camper, cargo, solar, antenna, armor, neon, mute).
- **Bottom**: radio dial + station marquee.

### Keyboard

- `C` — toggle cinematic (UI hidden)
- `P` — save a PNG screenshot of the current frame
